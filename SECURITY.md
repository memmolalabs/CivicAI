# Security Policy

## Reporting a Vulnerability

Do not post credentials, personal data, or immediately usable exploit details in open issues. Initially, use a private channel designated by the maintainer and coordinate disclosure.

## Security Properties v0.3.1

- no keys, tokens, or credentials in code;
- no telemetry or remote AI calls;
- no host permission or content scripts;
- untrusted text rendered via `textContent` or `value`, never as HTML;
- no `innerHTML`, `eval`, `new Function`, or remote code in CivicAI application code;
- MV3 CSP limited to local scripts and `wasm-unsafe-eval` required for WebAssembly;
- ONNX Runtime configured with `numThreads = 1`, SIMD enabled, and proxy disabled;
- no dependencies on `SharedArrayBuffer`, cross-origin isolation, or Blob workers;
- Assets downloaded to temporary files and accepted only after size and SHA-256 verification;
- Selections removed from local storage after panel consumption.

The third-party Transformers.js bundle incorporates generic compatibility code, including code not used in the CivicAI path. The MV3 CSP remains the execution authority; any bundle updates must be reviewed and hashes must be maintained.

## Data and Assets Not to Commit

- `.env`, keys, certificates, passwords, and tokens;
- Dumps, logs, or custom paths;
- ONNX models, runtime bundles, WASM files, and `SHA256SUMS.local.txt` generated by the script.

## Limitations

Local processing reduces data exposure but does not protect against a compromised device, untrusted extensions/browsers, or assets installed outside the verified script.
